# Plan 06-01 — Single Spread Renderer

## Objective
Given a spread definition from TODDLER_DATA and a set of photos (with detected orientations), render one complete spread (left + right pages) with photos placed at mm coordinates and SVG overlays on top.

## Context
- @assets/Template_Toddler/template-data.js — spread config (from 05-01)
- @pages/template-engine.html — shell (from 05-01)
- @pages/spread-preview.html — reference for EXIF reading, orientation detection, photo loading

## Coordinate system (critical)
- Page canvas = 600×600px
- Scale = 3px/mm (600 / 200)
- Photo slot: `left = x * 3`, `top = y * 3`, `width = w * 3`, `height = h * 3`
- SVG overlay: `<img src="...svg" style="position:absolute;top:0;left:0;width:100%;height:100%;pointer-events:none">`
- Photos use `object-fit: cover` inside sized divs

## Tasks

### Task 1 — EXIF reader + orientation detector
Copy/adapt from spread-preview.html:
- `readExifDate(file)` → ISO date string or null (uses exifr via CDN)
- `detectOrientation(img)` → 'H' or 'V' (based on rendered width vs height)
- `sortPhotosByDate(files)` → sorted array with objectURL, date, orientation

Attach to the Upload button in template-engine.html. On upload:
1. Read all files → EXIF dates → sort → detect orientations
2. Store as `window.photoPool = [{ file, url, date, orientation }]`
3. Render photo strip thumbnails (50px height, scrollable row)

### Task 2 — `renderSpread(spreadId, photoSlice)` function
Takes a spread ID (e.g. 'SP1') and a slice of photoPool entries.

Returns a DOM element: a `.spread-row` div containing:
- Left page div (or empty if SP0/rightOnly)
- Right page div

Each page div:
- 600×600px, position:relative, background:#FDF1E5 (Beige)
- Photo fills: absolutely positioned divs with `object-fit:cover` at slot coords
- SVG overlay: `<img>` tag pointing to `../assets/Template_Toddler/Spreads/{svg path}`
- Spread label above: "Spread N" or functional page name

Photo-to-slot assignment:
- Pick correct H or V variant based on photo orientation
- If 2 slots on a page: assign first and second photo in sequence
- If photo pool runs short: leave slot empty (show dashed placeholder)

### Task 3 — Wire up: render one test spread
On photo upload completion, render SP1 using the first photos in pool and append to `#book-canvas`. This is a smoke test — full book sequence comes in 06-02/06-03.
