# Plan 06-02 — Book Sequence Builder

## Objective
Given page count and selected functional pages, generate the ordered list of spread IDs that make up the complete book.

## Context
- @assets/Template_Toddler/template-data.js
- @pages/template-engine.html (with renderSpread from 06-01)
- @.planning/BRIEF.md — sequence rules

## Sequence rules
1. Spread 0 always first (right-only)
2. Standard spreads SP1–SP6 cycle for the remaining positions
3. Functional pages selected by staff are inserted among standards (evenly distributed by default)
4. Each spread = 2 pages except SP0 = 1 page
5. Total pages = pageCount → total spreads = (pageCount / 2) (SP0 counts as 1 page, so the math is: 1 + (pageCount - 1) / 2 spreads)

## Example: 20-page book, Birthday + Art selected
- 1 (SP0) + 9 remaining spreads = 10 spreads total (1 + 19 pages ÷ 2? No...)
- Actually: SP0 = 1 right page, then pairs. 20 pages total: 1 + 19? No — books must be even.
- Correct: 20 pages = 10 spreads. SP0 = pages 1. Then 9 more spreads = pages 2–20 (wait, 9×2=18, +1=19, not 20).
- Resolution: SP0 takes page 1 (right only). Remaining = 19 pages... but 19 is odd. 
- **Assumption:** Page count is always even (20, 28, 36...). SP0 occupies 2 physical pages (left is blank/cover back). So 20-page book = 10 spreads including SP0.
- Standard slots available: 10 - 2 (FPs) = 8. Cycle SP1–SP6 for 8: SP1,SP2,SP3,SP4,SP5,SP6,SP1,SP2.
- Insert FPs at evenly spaced positions: after position 3 and 6 (or similar).

## Tasks

### Task 1 — `buildBookSequence(pageCount, selectedFPs)` function
Returns an array of spread IDs in book order.

```js
// selectedFPs: array of FP IDs e.g. ['FP1', 'FP5']
// pageCount: even number, min 20, max 80, step 4 (enforced by config bar dropdown)
function buildBookSequence(pageCount, selectedFPs) → ['SP0', 'SP1', 'SP2', 'FP1', 'SP3', ...]
```

Page count math — explicit formula:
```
totalSpreads = pageCount / 2
// SP0 always at index 0, counts as 1 spread (occupies 2 physical pages: blank left + photo right)
// Example: 20-page book → totalSpreads = 10 → SP0 + 9 more spreads
remainingSpreads = totalSpreads - 1                   // 9
standardCount    = remainingSpreads - selectedFPs.length  // e.g. 9 - 2 = 7
```

Algorithm:
1. Total spreads = pageCount / 2
2. SP0 is always index 0
3. Remaining = totalSpreads - 1
4. Standard count = remaining - selectedFPs.length
5. Generate standard cycle: SP1, SP2, SP3, SP4, SP5, SP6, SP1, ... (standard count items)
6. Insert each FP at evenly spaced positions in the standard array
7. Return [SP0, ...interleaved]

### Task 2 — Photo pool allocator `assignPhotosToSpreads(sequence, photoPool)`
Populates `window.bookAssignments` for the entire book sequence.

Rules:
- Walk spreads in sequence order
- For each spread, determine orientation variant based on next photo's orientation (H→H variant, V→V variant)
- For each slot in that spread (left page first, then right page), assign the next unassigned photo from pool
- Photo-to-slot orientation matching: select the spread variant (H/V) that matches the next photo's orientation; the variant is chosen once per page based on the first photo assigned to that page
- Special slots (FP left text pages, FP3/FP4 special uploads, FP5 artwork) get null — handled separately in 08-xx
- Track which photos are consumed; unassigned photos shown in photo strip with warning
- Stored as pool indices in window.bookAssignments (not photo objects): bookAssignments[spreadIndex].left[0] = 5

### Task 3 — Wire to config bar
When user changes page count or FP checkboxes:
1. Rebuild sequence with `buildBookSequence`
2. Re-allocate photos with `assignPhotosToSpreads`
3. Re-render book canvas (clear + re-render all spreads)

## Verify
- 20-page book with no FPs: sequence is SP0, SP1, SP2, SP3, SP4, SP5, SP6, SP1, SP2, SP3 (10 spreads)
- 20-page book with Birthday + Art: FPs inserted at roughly 1/3 and 2/3 positions
- Adding/removing FP checkbox re-renders sequence in canvas
- Photo count warning shown if more photos than slots or vice versa
