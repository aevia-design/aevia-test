# Plan 11-01 — Photo Count Calculator on scribble.html

## Objective
Show customers a live "you'll need approximately X–Y photos" count on the scribble.html product page, updating as they change page count and FP addon selections. Also align the accepted file formats and guidance copy with what template-engine.html actually accepts.

## Context
- @pages/scribble.html — product page; page count chips + FP addon chips already wired; `goToOrder()` passes selections to order.html
- @pages/template-engine.html — reference for photo slot counting logic and accepted formats
- @assets/Template_Scribble/template-data.js — spread definitions; slot counts per spread type/variant

## Photo count logic (replicate from template engine)

The number of photo slots depends on:
1. Page count (40 or 80 pages → 20 or 40 spreads)
2. Which FP addons are selected (FP1–FP5 replace standard spreads; each FP has its own slot count)
3. Spread variant (L2, L3, L4, etc.) — customer doesn't control this; use the **range** across all variants

**Algorithm:**
- SP0 (cover) always contributes 1 slot
- For each non-FP spread position, look up all variants in SCRIBBLE_DATA and count slots with `pool: 'regular'`. Min and max across variants = the range contribution.
- For each selected FP, use that FP's slot count (sum across left+right pages, only `pool: 'regular'` or `pool: 'special'` slots that require a customer photo).
- Special-pool slots (FP birthday photo, toy, steps, artwork) are counted separately as "special photos" — they don't come from the main pool.
- Sum min slots and max slots separately to get range.

**Practical approach:** Rather than loading template-data.js into scribble.html (it's large), hardcode a lookup table derived from it. This table should be recalculated from template-data.js if spreads change.

Hardcode in scribble.html as a const:
```js
// Main pool slots per spread type, [min, max] across variants
const SPREAD_SLOT_RANGE = {
  SP0: [1, 1],   // cover, always 1
  SP1: [2, 4],   // standard spread — check actual template-data values
  SP2: [2, 4],
  // ... all SP types
  FP1: [0, 0],   // FP1 main pool slots (birthday photo is special pool)
  FP2: [0, 0],   // FP2 has no photo slots (text only)
  FP3: [0, 0],   // toy photo is special pool
  FP4: [0, 0],
  FP5: [0, 0],
};

// Special pool slots (uploaded separately in order form)
const FP_SPECIAL_SLOTS = { FP1: 1, FP3: 1, FP4: 1, FP5: 1 }; // FP2 = 0 (text only)
```

**Before writing this plan's code:** read template-data.js and count the actual `pool:'regular'` slots per spread type per variant to fill in the correct values. Do not guess.

**Book sequence for photo count:**
- 40 pages = 20 spreads: SP0 (cover) + 19 content spreads
- 80 pages = 40 spreads: SP0 + 39 content spreads
- FP spreads replace content spread positions (not cover)
- Standard spread type distribution: use the most common variants from `buildBookSequence()` logic in template-engine — or simply use SP1 as the baseline standard spread if the exact distribution isn't available

## Tasks

### Task 1 — Read template-data.js and derive lookup table
Read `assets/Template_Scribble/template-data.js`. For each spread type (SP0–SP14, FP1–FP5):
- Count slots with `pool: 'regular'` in each variant (H and V for left and right pages)
- Record [min, max] across all variants
- Count `pool: 'special'` slots (these are FP-specific uploaded photos)
- Write the completed `SPREAD_SLOT_RANGE` and `FP_SPECIAL_SLOTS` lookup tables

### Task 2 — Live counter function in scribble.html
Add function `updatePhotoCount()` that:
1. Reads currently selected page count (40 or 80) from the active chip
2. Reads selected FP addons from `.addon.on` elements → their `data-fp` attributes
3. Calculates book sequence: SP0 + (totalSpreads - 1 - fpCount) standard spreads + selected FPs
4. Sums slot ranges to get [minPhotos, maxPhotos]
5. Updates a visible counter element

Counter display location: below the FP addon section, above the CTA button. 
Copy: **"You'll need [min]–[max] photos"** in the same style as existing hint text.
If FPs include text-only (FP2): counter copy adds **"+ your Funny Words list"** as a note beneath.
Special photos (FP1/3/4/5): counter copy adds **"+ [N] dedicated spread photo(s)"**.

Call `updatePhotoCount()` on: page load, every chip click, every addon toggle.

### Task 3 — Accepted formats note
Below the photo count, add a single line in muted style:
**"We accept JPEG, PNG, and HEIC. RAW files (DNG, CR2, NEF, etc.) must be exported as JPEG first."**

This matches exactly what template-engine.html enforces.

- Toggling FP2 on: "+ your Funny Words list" appears
