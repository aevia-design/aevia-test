# Plan 11-02 ‚Äî `getOrderAssets` Cloud Function

## Objective
Add a new Firebase Cloud Function that staff can call from template-engine.html with an order number. It returns the full order metadata from Firestore plus signed GCS read URLs for every uploaded file in that order's folder.

## Context
- @functions/index.js ‚Äî existing function entry points (`createUploadSession`, `generateCaption`)
- @functions/upload.js ‚Äî order creation logic; shows Firestore document structure and GCS folder naming
- @functions/package.json ‚Äî dependencies already installed
- @functions/.env ‚Äî Gmail credentials (reference only, do not change)
- Firestore collection: `orders` (confirmed in upload.js)
- GCS bucket: `aevia-uploads.firebasestorage.app`
- Firebase project: `aevia-uploads`, region: `europe-west1`

## Firestore order document structure (from upload.js)
Key fields expected:
- `orderNumber` ‚Äî e.g. `AEV-001`
- `folderName` ‚Äî GCS folder where photos live
- `templateName` ‚Äî e.g. `Scribble`
- `pageCount` ‚Äî `40` or `80`
- `specialRequests` ‚Äî free text; contains FP selections list + FP2 funny words + customer notes
- `customerName`, `email`, `status`, `createdAt`

Note: FP selections are stored as text in `specialRequests` (e.g. "Add-ons selected: Birthday spread, Funny words\n\nFunny Words: Noooo, More!, Biiiig"). Parse these out in the function response.

### Task 2 ‚Äî Implement `getOrderAssets` function
Add to `functions/index.js` (or a new `functions/assets.js` imported there):

```js
exports.getOrderAssets = functions.region('europe-west1').https.onRequest(async (req, res) => {
  // CORS headers (same pattern as existing functions)
  // Accept POST only
  // Body: { orderNumber: 'AEV-001' }
  
  // 1. Look up Firestore order by orderNumber field
  // 2. If not found: 404 { error: 'Order not found' }
  // 3. Parse specialRequests to extract:
  //    - fpSelections: string[] (e.g. ['Birthday spread', 'Funny words'])
  //    - fp2Words: string | null (comma-separated funny words if present)
  //    - customerNotes: remaining text after stripping addon lines
  // 4. List all files in GCS folder (bucket.getFiles({ prefix: folderName }))
  // 5. For each file: generate signed GET URL (expires 1 hour)
  //    Include metadata: { name, url, fileType, addonSlug }
  //    - fileType comes from the file's GCS metadata (set during upload in upload.js)
  //      OR inferred from filename prefix if metadata not available
  // 6. Return:
  {
    order: {
      orderNumber, customerName, email, templateName, pageCount,
      fpSelections,   // string[] parsed from specialRequests
      fp2Words,       // string | null
      customerNotes,  // remaining notes
      status, createdAt
    },
    files: [
      { name: 'cover.jpg', url: 'https://storage.googleapis.com/...', fileType: 'cover', addonSlug: null },
      { name: 'fp1_birthday.jpg', url: '...', fileType: 'special', addonSlug: 'fp1' },
      { name: 'photo_001.jpg', url: '...', fileType: 'main', addonSlug: null },
      // ...
    ]
  }
});
```

**Parsing `specialRequests`:**
The format written by order.html is:
```
Add-ons selected: Birthday spread, Funny words

Funny Words: Noooo, More!, Biiiig

[customer free-text notes]
```
Parse with string splitting on newline + known prefixes. Do not use regex if simple split works.

**File metadata:** During `createUploadSession`, each file has `fileType` and `addonSlug` in the request body (`files[]` array). Check if these are stored as GCS object metadata. If yes, read them. If not, infer from filename patterns (cover file is always first, filenames may include 'cover', 'fp1', etc.).
