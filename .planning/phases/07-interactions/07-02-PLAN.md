# Plan 07-02 — Spread Reorder + Swap Type

## Objective
Staff can reorder the book sequence (move a spread up/down) and change a spread's type (e.g. swap SP2 for SP4, or replace a standard spread with a functional one).

## Context
- @pages/template-engine.html (with book sequence from 06-02)

## Tasks

### Task 1 — Move spread up/down
Each spread row has `[↑]` and `[↓]` buttons:
- Swap this spread's position with adjacent spread in `window.bookSequence`
- SP0 (index 0) cannot be moved — hide buttons on it
- Re-render only the two affected spread rows after swap
- Update spread index labels

### Task 2 — Change spread type dropdown
Each spread row has a `[Change type ▼]` dropdown showing:
- All standard spread IDs: SP1, SP2, SP3, SP4, SP5, SP6
- All functional spread IDs currently selected in config: FP1, FP2, etc.
- Current type shown as selected/disabled
- On select: replace this position in `bookSequence` with chosen type, re-render the row

**Constraint:** A functional spread can appear in multiple positions (user might want Birthday wishes early and late), and standard spreads can repeat freely. No uniqueness enforcement needed — staff decides.

**Photo reassignment on type change:** When a spread changes type, its current photo assignments are cleared (photos return to unassigned pool). The new spread type gets its slots filled from pool tail. Photo status bar updates.

### Task 3 — Visual sequence indicator
Below the config bar (above photo strip): a compact horizontal strip showing the book sequence as pill badges:
```
[SP0] [SP1] [SP2] [FP1★] [SP3] [SP4] [FP5★] [SP5] [SP6] [SP1]
```
Functional pages shown with ★. Clicking a pill scrolls to that spread. Updates whenever sequence changes.

## Verify
- Moving spread 3 up swaps with spread 2, index labels update
- Changing SP2 → SP4 in position 2: spread re-renders with SP4 layout
- Replacing a standard with a functional: text/special photo placeholder appears
- Sequence pill strip updates on every reorder/swap
- SP0 move buttons are hidden
