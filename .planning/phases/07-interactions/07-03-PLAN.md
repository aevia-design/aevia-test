# Plan 07-03 — Caption Layer

## Objective
Each photo slot that has `caption: allowed` shows an editable caption text overlay positioned exactly per the Excel spec. Staff type captions inline; an AI button generates a suggestion.

## Context
- @assets/Template_Toddler/template-data.js — caption position rules per slot
- @pages/spread-preview.html — reference for caption rendering and AI button
- @pages/template-engine.html

## Caption position rules (from Excel)
```
'below'       → caption text starts Nmm below photo bottom edge
'above'       → caption text starts Nmm above photo top edge  
'upper-right' → caption text placed Nmm to the right of the photo's top-right corner, horizontal (NOT rotated)
'center'      → centered on the full page (for text pages FP1/FP2)
```

"Upper-right" is confirmed horizontal text — no rotation. It applies to SP0 Right V and SP3 Right V (portrait photo spreads).
Formula: `top = y * 3`, `left = (x + w + offset) * 3`, `width = (pageSize - x - w - offset) * 3` (remaining page width).
Note: FP1 Birthday right page (heart photo) has NO caption — the upper-right slots are only on SP0/SP3.

## Tasks

### Task 1 — Caption overlay rendering
For each slot where `caption.allowed = true`, add a `.caption-overlay` div:
- Position calculated from slot coords + caption position rule + offset (all in mm → px):
  - `below`: `top = (y + h + offset) * 3`, `left = x * 3`, `width = w * 3`
  - `above`: `top = (y - offset - captionHeight) * 3`, same left/width
  - `upper-right`: `top = y * 3`, `left = (x + w + offset) * 3`, width = remaining space to page edge
  - `center`: centered on 600px page, `top = 50%`, `left = 50%`, `transform: translate(-50%,-50%)`
- Style: font-family NT Comic / EB Garamond (per template-info), color #493955 (Plum), font-size ~11px, text-align center
- `contenteditable="true"` on the caption div for inline editing
- Placeholder text "Add caption..." (shown in lighter color via CSS ::before when empty)

### Task 2 — AI caption button
Each caption-allowed slot gets a small `[✦ Generate]` button adjacent to its caption overlay.
- On click: call existing `/api/generate-caption` or the Firebase caption function
- Pass the photo's objectURL as base64 or the slot image src
- On response: fill the caption div with suggested text
- Button shows spinner while loading, error state if call fails

### Task 3 — Caption state persistence
Store captions in `window.captionState[spreadIndex][page][slotIndex] = 'text'`.
On re-render of a spread row (after drag/reorder), restore caption text from this state.
Caption text is NOT lost when spread row is re-rendered.

## Verify
- SP1 left page shows caption overlay below photo (at 10mm or 6mm per orientation)
- SP3 right page shows caption overlay below photo (at 50mm)
- SP5 right page shows caption overlay ABOVE photo
- Caption is editable — typing updates the text on the page
- Re-ordering spreads preserves caption text
- AI button calls caption function and fills text
