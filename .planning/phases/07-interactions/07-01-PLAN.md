# Plan 07-01 — Photo Slot Drag (Within + Across Spreads)

## Objective
Staff can drag a photo from any slot and drop it into any other slot anywhere in the book. The template re-renders the affected spreads automatically.

## Context
- @pages/template-engine.html (with full book rendered from 06-03)

## Design
- Drag source: any filled photo slot div
- Drop target: any photo slot div (filled or empty) in any spread
- On drop: swap photo assignments between source and target slots
- Re-render only affected spread rows (not full book re-render — too slow)
- Visual feedback: drag ghost = photo thumbnail, drop target highlights on hover

## Tasks

### Task 1 — Slot drag-and-drop system
Using HTML5 drag-and-drop API (same approach as spread-preview.html):
- Each photo slot div gets `draggable="true"` when it contains a photo
- `dragstart`: store slot identity (spreadIndex, page, slotIndex) in dataTransfer; also store pool type ('regular' | 'special-FP3' | 'special-FP4' | 'special-FP5' | 'artwork-FP5')
- `dragover` on any slot: `preventDefault()`, add `.drop-target` highlight class
- `drop`: validate pool types match before swapping — see constraint below

Drop validation (enforce this strictly):
```
ALLOW drop if:
  - Both source and target are 'regular' pool slots

BLOCK drop if:
  - Source is any special/artwork pool type → dropping to a different slot type
  - Target is a special upload slot (FP3 left, FP4 left, FP5 left/right)
  → Show no visual feedback (drop rejected silently or with a shake animation)
```
Special photos (FP3, FP4, FP5) can only be replaced via the special upload zones, not by drag-and-drop.

State model: `window.bookAssignments[spreadIndex][page][slotIndex] = photoPoolIndex | null`
Swap = swap two integer indices (never duplicate or lose references).

### Task 2 — Targeted re-render
`rerenderSpread(spreadIndex)` function:
- Remove existing `.spread-row` at that index from DOM
- Re-create it with updated assignments
- Insert at same position in canvas

Avoids re-rendering all 10+ spreads on every drag.

### Task 3 — Empty slot as drop target
Empty slots (placeholder dashed border) should also accept drops — photo moves from source slot to empty slot (source becomes empty).

## Verify
- Drag photo from SP1 left slot → drop to SP2 right slot: photos swap
- Drag photo to empty slot: photo moves, source becomes empty placeholder
- Correct SVG variant selected after re-render (H/V based on photo orientation)
- Undo is NOT required (staff can drag back)
