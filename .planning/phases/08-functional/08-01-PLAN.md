# Plan 08-01 — Functional Page Text Panels

## Objective
FP1 (Birthday wishes) and FP2 (Funny words) have a left page that is pure text — no photo slot. Render styled text panels for these, editable inline. Right pages render normally (photo + SVG clip).

## Context
- @assets/Template_Toddler/template-data.js — FP1 and FP2 definitions
- @assets/Template_Toddler/Spreads/FP Spread 1/ and FP Spread 2/ — SVG files for both pages

## FP1 — Birthday wishes
- Left page: large decorative text field. Style: NT Comic font, centered, warm color (Coral #F47E67 or Plum).
  User-entered birthday message goes here. Staff types it in. No character limit enforcement — overflow clips.
- Right page: heart-shaped photo. The SVG (`FP Birthday 02 R copy.svg`) contains the heart clip path.
  Photo goes behind the SVG — the clip path in the SVG masks it. This works automatically if the SVG is layered on top.

## FP2 — Funny words
- Left page: same as FP1 left — text field for the child's funny phrases/words. 
- Right page: full-bleed square photo (200×200mm = 600×600px, covers entire page). 
  Photo fills the full page div. SVG overlay on top for decorative elements.

## Tasks

### Task 1 — Text page renderer `renderTextPage(fpId, pageType)`
Returns a 600×600px page div containing:
- Background: Beige (#FDF1E5)
- SVG overlay (left page SVG for that FP) on top
- Central text area: `<div contenteditable="true" class="fp-text-panel">` 
  - Font: NT Comic, ~20px, color: Plum
  - Centered vertically and horizontally within the page
  - Placeholder text per FP: "Birthday message..." / "Funny words and phrases..."
  - Restore text from `window.captionState` on re-render

### Task 2 — Full-bleed right page for FP2
FP2 right page: photo fills the entire 600×600 page div (no slot inset).
- `window.bookAssignments` still tracks which photo is assigned to "FP2 right"
- Photo div: `width: 100%; height: 100%; object-fit: cover; position: absolute; top: 0; left: 0`
- SVG overlay on top
- Drag-and-drop works same as regular slots (07-01)

### Task 3 — Heart photo for FP1 right
FP1 right page: the heart clip IS in the SVG. So:
- Photo placed at slot coords from data: x=105, y=100, w=165, h=175 (mm → px)
- SVG placed on top — its clip paths mask the photo into heart shape automatically
- The photo div needs `z-index` below the SVG; the clip-path region in the SVG is transparent
- Test: upload a photo to FP1 right slot and confirm heart shape is visible

## Verify
- FP1 left shows a styled editable text panel with NT Comic font
- FP1 right shows a heart-masked photo (photo visible through heart shape in SVG)
- FP2 left shows an editable text panel
- FP2 right shows full-bleed photo covering entire page
- Typing in text panel persists through spread re-renders
