# Plan 01-03 — Spread Renderer (Photo Placement)

## Objective
Render the assembled spread in the browser. Two 200×200mm page boxes side by side, photos placed at exact spec coordinates using absolute positioning. After this plan you can see the actual photos sitting in their correct slots.

## Context
- Builds on 01-02 (variant picker working, photos sorted + oriented)
- Each page = 200mm × 200mm. Render at screen scale: 1mm = 3px → each page = 600×600px on screen
- Coordinates in SPREAD_SPEC = center of slot in mm. Convert to px top-left:
  `left = (cx - w/2) * scale`, `top = (cy - h/2) * scale`
- Bleed = 3mm. The spec coords are WITHOUT bleed. For screen preview, ignore bleed offset (it's only relevant for print export)
- Photos placed with `object-fit: cover` to fill slot without distortion

## Scale constant
```js
const MM_TO_PX = 3; // 1mm = 3px → 200mm page = 600px
```

## Tasks

### Task 1 — Build `renderSpread(photos, leftVariant, rightVariant)` function

Function receives:
- `photos`: array of 3 objects `{ file, objectURL, orientation, date }`
- `leftVariant`: 'L1' or 'L2'
- `rightVariant`: 'R1', 'R2', or 'R3'

Function builds and injects into `#spread-container`:

```
#spread-container
  .spread (display:flex, gap:4px)
    .page.page-left  (600×600px, position:relative, background:#f5f0eb)
      .photo-slot (position:absolute, overflow:hidden, per slot in leftVariant)
        img (width:100%, height:100%, object-fit:cover)
      .svg-overlay (position:absolute, inset:0, pointer-events:none) ← empty for now
    .page.page-right (600×600px, position:relative, background:#f5f0eb)
      .photo-slot × N (based on rightVariant slot count)
        img
      .svg-overlay ← empty for now
```

Photo assignment:
- Left page slot 0 ← photos[0]
- Right page slot 0 ← photos[1]
- Right page slot 1 ← photos[2] (if rightVariant has 2 slots)

Slot positioning CSS (calculated from spec):
```js
slot.style.left   = (slotDef.cx - slotDef.w / 2) * MM_TO_PX + 'px';
slot.style.top    = (slotDef.cy - slotDef.h / 2) * MM_TO_PX + 'px';
slot.style.width  = slotDef.w * MM_TO_PX + 'px';
slot.style.height = slotDef.h * MM_TO_PX + 'px';
```

### Task 2 — Call renderSpread after variant picker

In the file input `change` handler (from 01-02), after picking variants:
1. Create objectURLs for each file: `URL.createObjectURL(file)`
2. Call `renderSpread(photos, leftVariant, rightVariant)`

### Task 3 — Visual polish
- Page background: `#f5f0eb` (Aevia off-white)
- Thin 1px border on each page box (for layout reference, can remove later)
- `#spread-container` has top margin and is centered
- Photo slot placeholder (before photos load): light grey background with dashed border
