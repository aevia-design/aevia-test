### Task 1 — SVG overlay layer
In `renderSpread()`, after placing photos, add the overlay `<img>` (or `<object>`) to each page:

```js
function addOverlay(pageEl, variantName) {
  const path = `../assets/Template/Spreads/${variantName}.svg`;
  const overlay = document.createElement('img');
  overlay.src = path;
  overlay.className = 'svg-overlay';
  overlay.onerror = () => {
    // SVG not yet available — show placeholder outline
    overlay.style.border = '2px dashed #aaa';
    overlay.style.opacity = '0.4';
  };
  pageEl.appendChild(overlay);
}
```

CSS for `.svg-overlay`:
```css
.svg-overlay {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  pointer-events: none;
  z-index: 10;
}
```

Call `addOverlay(leftPageEl, leftVariant)` and `addOverlay(rightPageEl, rightVariant)`.

### Task 3 — Manual photo reorder
Above the spread, show a row of 3 numbered photo thumbnails (the current order):

```
[1] thumb  [2] thumb  [3] thumb    [Re-sequence ↺]
```

Each thumbnail has up/down arrows (← →) to swap adjacent photos.
On swap: update the internal `photos` array order and re-call `renderSpread()`.

"Re-sequence" button: re-runs EXIF date sort (resets to original chronological order).

Implementation: simple array swap, no drag-drop library needed for MVP.
