# Plan 01-02 — EXIF Reading + Orientation + Variant Picker

## Objective
Wire up the file input. When 3 photos are selected: read EXIF dates, detect orientations, sort chronologically, pick the correct L and R variants. Log results to console. After this plan the logic is solid before any rendering.

## Context
- Builds on 01-01 (page shell exists, SPREAD_SPEC is defined)
- No server — all JS runs in browser
- EXIF reading: use `exifr` library via CDN (small, browser-compatible, reads DateTimeOriginal)
- Orientation detection: from EXIF `Orientation` tag AND from image natural dimensions as fallback
- Photos in: `assets/test-photos/` (3 JPEGs user drops there for testing)

## Variant selection logic
```
Photo 1 (earliest EXIF date) → Left page
  horizontal → L1
  vertical   → L2

Photos 2 + 3 → Right page
  both horizontal → R1
  both vertical   → R2
  mixed           → R3
```

## Tasks

### Task 1 — Load exifr via CDN
Add to `<head>` of spread-preview.html:
```html
<script src="https://cdn.jsdelivr.net/npm/exifr/dist/lite.umd.js"></script>
```

### Task 2 — Wire file input → EXIF pipeline
In the `<script>` block, add a `change` listener on the file input that:

1. Reads all selected files
2. For each file, calls `exifr.parse(file, ['DateTimeOriginal', 'Orientation', 'PixelXDimension', 'PixelYDimension'])` 
3. Determines orientation for each photo:
   - Primary: from EXIF `Orientation` tag (1-8). Values 5,6,7,8 = portrait, others = landscape
   - Fallback: create an Image object, read naturalWidth vs naturalHeight. taller = portrait.
4. Sorts photos by `DateTimeOriginal` ascending (earliest first)
5. Assigns: photo[0] → left page, photo[1] and photo[2] → right page
6. Picks L variant: photo[0].orientation === 'vertical' ? 'L2' : 'L1'
7. Picks R variant based on photo[1] + photo[2] orientations:
   - both horizontal → 'R1'
   - both vertical → 'R2'
   - mixed → 'R3'
8. Logs to console: sorted order, each photo's date + orientation, chosen L variant, chosen R variant

### Task 3 — Handle edge cases gracefully
- Fewer than 3 photos selected: show message "Please select exactly 3 photos"
- Photo has no EXIF date: fall back to file.lastModified date, log a warning
- All 3 same date: keep file order as tiebreaker
