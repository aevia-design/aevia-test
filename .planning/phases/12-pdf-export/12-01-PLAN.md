# Plan 12-01 — High-Quality PDF Export (Puppeteer)

## Objective
Staff clicks "Export for print" in template-engine.html → downloads a print manifest JSON → runs a local Node.js script that generates a print-quality PDF at 300 DPI equivalent using Puppeteer headless Chrome.

## ⚠ Prerequisites
1. Plan 11-03 complete (photos have GCS signed URLs, not just blob URLs)
2. User confirms bleed/mark requirements before implementation begins

**STOP — before implementing this plan, ask the user:**
- Trim size confirmation: 200mm × 200mm content, 3mm bleed all sides → 206mm × 206mm total. Correct?
- Crop marks: required in PDF? If yes: inside bleed or outside? What weight/length?
- Color profile: sRGB or CMYK? (Puppeteer outputs sRGB; CMYK conversion may need separate tool)
- PDF version requirement from the print house? (PDF/X-1a, PDF/X-4, or standard PDF 1.4+)
- File naming convention? (`AEV-001-print.pdf`?)
- Should spine be a separate page or part of a cover spread page?

Do not proceed with implementation until these are answered.

## Architecture

```
template-engine.html
  → "Export for print" button
  → generates print manifest JSON
  → triggers download of manifest.json

staff runs locally:
  node export.js --manifest path/to/manifest.json --output AEV-001.pdf

export.js:
  → reads manifest
  → launches Puppeteer headless Chrome
  → loads print-render.html with manifest injected
  → renders each page at deviceScaleFactor:6, @page 206mm×206mm
  → exports PDF
```

## Print manifest structure
```json
{
  "orderNumber": "AEV-001",
  "templateName": "Scribble",
  "pageCount": 40,
  "spreads": [
    {
      "index": 0,
      "type": "SP0",
      "variant": { "left": null, "right": "H" },
      "photos": {
        "right": [{ "url": "https://storage.googleapis.com/...", "x": 105, "y": 70, "w": 150, "h": 100 }]
      },
      "captions": {
        "right": { "0": { "text": "Summer 2024", "style": { "fontFamily": "Playfair Display", "fontSize": "14pt", "textAlign": "center" } } }
      }
    }
  ],
  "cover": {
    "bgColor": "#f5e6d0",
    "photos": { ... },
    "captions": { ... }
  }
}
```

Photo URLs must be GCS signed URLs (from Plan 11-02/11-03), not blob: URLs. Blob URLs are not accessible by Puppeteer.

## Tasks

### Task 1 — Print manifest export in template-engine.html
Add "Export for print" button to the engine UI (near the top, styled as a secondary action button).

On click:
1. Collect current state: `window.bookSequence`, `window.bookAssignments`, `window.bookCaptions`, cover config
2. For each spread + each slot: include photo URL (from `window.photos[idx].url` — this must be the GCS URL if loaded via Plan 11-03, or a base64 data URL if photos were manually uploaded)
3. Warn user if any photos are blob: URLs (manually uploaded, not from GCS): "Some photos are from local uploads and will be embedded as base64 — manifest will be large."
4. For blob: URL photos: convert to base64 via canvas (reuse `urlToBase64()`)
5. JSON.stringify manifest → create Blob → trigger download as `manifest-AEV-XXX.json`

### Task 3 — `print-render.html`
Create `pages/print-render.html` — a stripped-down version of the template engine that:
- Accepts `?spread=N` query param
- Reads `window.__MANIFEST` (injected by Puppeteer via `page.evaluate`)
- Renders exactly one spread at SCALE=3 (matching engine preview)
- `deviceScaleFactor:6` in Puppeteer compensates to reach effective 18px/mm = 457 DPI
- No UI chrome — just the spread canvas, full-bleed
- Adds class `spread-ready` to body when render complete (Puppeteer waits for this)
- `@page { size: 206mm 206mm; margin: 0; }`

This file reuses rendering functions from template-engine but is self-contained (copy the minimum needed, do not import from template-engine.html).

## Verify
- Export button downloads a valid JSON manifest
- Manifest contains correct spread sequence, photo URLs, captions
- `node export.js --manifest manifest.json` runs without error
- Output PDF opens in Preview/Acrobat: correct page size (206×206mm)
- Photos render sharp (not pixelated) — at deviceScaleFactor:6, a 200mm slot = ~2268px output
- Captions render with correct fonts, sizes, positions
- SVG overlays appear correctly
- Page count matches book (20 pages for 40-page book, or 40 pages for 80-page book)
- checkpoint:human-verify — user opens PDF, sends one page to print house for test print, confirms bleed and resolution acceptable
