## Tasks

### Task 0 — Define global state model (set this up in template-engine.html, not template-data.js)
Before any rendering, initialise these globals. All other phases read/write to these exact shapes.

```js
// Photo pool — filled when user uploads files
window.photoPool = [];
// Each entry: { file, url, date, orientation }
// url = objectURL (revoke old ones on re-upload)
// date = EXIF date or file.lastModified fallback
// orientation = 'H' (landscape) | 'V' (portrait) | 'S' (square)

// Assignment state — maps each slot to a photoPool index
window.bookAssignments = {};
// Shape: { [spreadIndex]: { left: [poolIndex|null, ...], right: [poolIndex|null, ...] } }
// Example: bookAssignments[2].right[0] = 5  → slot 1 on right page of spread index 2 = photo #5
// Swap: just swap two integers (e.g. bookAssignments[a].left[0] ↔ bookAssignments[b].right[0])
// On spread type change: set all assignments for that spreadIndex to null (photos return to unassigned pool)

// Caption state — text typed by staff per slot
window.captionState = {};
// Shape: { [spreadIndex]: { left: { 1: 'text', 2: 'text' }, right: { 1: 'text' },
//                           leftPanel: 'FP text content', rightPanel: '' } }
// 'leftPanel'/'rightPanel' used only for FP1/FP2 text-only pages
// All keys are strings; missing key = empty caption

// Special photo pools — for FP3, FP4, FP5
window.specialPhotos = { FP3: [], FP4: [], FP5: [] };
// FP3[0] = toy photo, FP4[0] = steps photo, FP5[0] = left artwork, FP5[1] = right artwork
// Each entry same shape as photoPool entry: { file, url, date, orientation }
```

### Task 1 — Create `assets/Template_Toddler/template-data.js`
A plain JS file (no module syntax — loaded with `<script src="">`) that sets `window.TODDLER_DATA`.

Structure:
```js
window.TODDLER_DATA = {
  template: 'toddler',
  pageSize: 200,          // mm, square
  bleed: 3,               // mm
  canvasPx: 600,          // render size per page
  scale: 3,               // px per mm (canvasPx / pageSize)
  fonts: { display: 'NT Comic', body: 'EB Garamond' },
  colors: { plum: '#493955', beige: '#FDF1E5', coral: '#F47E67' /* etc */ },
  
  spreads: {
    // Standard spreads (type: 'standard')
    SP0: { type: 'standard', id: 'SP0', label: 'Cover', rightOnly: true, pages: { right: { H: {...}, V: {...} } } },
    SP1: { type: 'standard', id: 'SP1', label: 'Spread 1', pages: { left: { H: {...}, V: {...} }, right: { H: {...}, V: {...} } } },
    // ... SP2–SP6
    
    // Functional spreads (type: 'functional')
    FP1: { type: 'functional', id: 'FP1', label: 'Birthday wishes', textLeft: true, pages: {...} },
    FP2: { type: 'functional', id: 'FP2', label: 'Funny words', textLeft: true, pages: {...} },
    FP3: { type: 'functional', id: 'FP3', label: 'Favourite toy', specialLeft: true, pages: {...} },
    FP4: { type: 'functional', id: 'FP4', label: 'First steps', specialLeft: true, pages: {...} },
    FP5: { type: 'functional', id: 'FP5', label: 'Art gallery', allArtwork: true, pages: {...} },
  }
};
```

Each page variant (H/V) contains:
```js
{
  svg: 'SP Spread 1/SP 01 H Left.svg',   // relative to assets/Template_Toddler/Spreads/
  slots: [
    {
      slot: 1,
      x: 95, y: 100, w: 150, h: 100,    // mm, content area coords
      caption: { allowed: true, position: 'below', offset: 10 }
      // position: 'below' | 'above' | 'upper-right' | 'center' | null
      // offset: mm from photo edge
    }
  ]
}
```

Use ALL the data from the Excel (above) — every spread, every slot, every caption rule.

### Task 2 — Create `pages/template-engine.html` shell
A new standalone page. Copy nav/footer from `pages/home.html`. Style matches Aevia brand.

Layout (top to bottom):
1. **Config bar** (sticky top strip):
   - "Page count:" dropdown (20, 28, 36, 44, 52, 60, 68, 76, 80 — common photobook sizes in multiples of 4 starting at 20)
   - "Functional pages:" 5 checkboxes — Birthday wishes / Funny words / Favourite toy / First steps / Art gallery
   - "Upload Photos" button (triggers hidden file input, accepts jpg/jpeg/heic/heif, multiple)
2. **Photo strip** (hidden until photos uploaded): horizontal scrollable row of photo thumbnails in sequence order
3. **Book canvas** (main area): empty div with id="book-canvas" and placeholder text "Upload photos and configure the book above to begin"

No rendering logic in this task — just the HTML/CSS shell.

Load `template-data.js` in the `<head>` (before closing body).
