# Plan 10-01 — Cover Renderer Audit + Fixes

## Objective
Audit the cover renderer in `template-engine.html`, document every visual bug or misalignment, and fix them in one pass. Cover = front canvas + spine + back, rendered as a single horizontal strip.

## Context
- @pages/template-engine.html — full engine; cover renderer is the `renderCover()` function and its surrounding HTML/CSS
- @assets/Template_Scribble/template-data.js — cover section data (bgColor, svg paths, caption positions)
- @assets/Template_Scribble/ — SVG files used in cover

## Known history (do not re-introduce these bugs)
- Cream strip between sections: was caused by per-div backgrounds. Fixed with single CSS `linear-gradient` on canvas element. Do NOT revert to div-per-section.
- Caption vertical centering: fixed with `top: capCY; transform: translateY(-50%)`. Do NOT change this.
- Spine caption: `top: capCY - lineHpx/2` where `lineHpx = sizePt * (4/3) * 1.1`. Do NOT change this.
- Line height: `sizePt * 1.1`. Do NOT change.
- EXIF Orientation swap: must NOT be applied. Browser auto-rotates.

## Tasks

### Task 1 — Visual audit (read + screenshot inventory)
Open `http://localhost:8080/pages/template-engine.html` with a set of test photos loaded and cover configured. Work through this checklist and note every issue found:

1. Front canvas: does `bgColor` apply correctly across the full front area?
2. Spine: correct width? (spine width = page count × 0.085mm × SCALE, minimum 5mm)
3. Back: background color matches spec?
4. SVG overlays: all three SVGs load, z-index correct (above photo, below captions)?
5. Cover photo: fills front slot without overflow outside canvas?
6. Front captions: positioned, sized, and centered correctly for 1-line and 2-line text?
7. Spine caption: centered vertically and horizontally in spine strip? Rotated 90°?
8. Back captions/text: positioned correctly?
9. Gradient boundary: no visible seam between front/spine/back sections?
10. At different `bgColor` values: does the whole front change correctly?

Document every bug found as a numbered list before making any code changes.

### Task 2 — Fix all identified bugs
For each bug from Task 1, apply the minimal targeted fix. Each fix should be one surgical Edit — do not refactor surrounding code unless the bug requires it.

After all fixes, re-run the Task 1 checklist mentally and confirm each item passes.

### Task 3 — Spine width calculation
Verify the spine width formula is implemented correctly:
- Formula: `spineWidthMm = pageCount * 0.085` (standard paper 115g/m²)
- Minimum spine: 5mm
- At SCALE=3: `spineWidthPx = Math.max(spineWidthMm, 5) * 3`
- 40 pages → 3.4mm → rounds up to 5mm minimum → 15px
- 80 pages → 6.8mm → 20.4px → ~20px

If spine width is hardcoded or wrong, fix it to use page count from `ORDER.pages` (or equivalent config variable).

## Verify
- Cover renders with no visible seams or color bleeds between sections
- Spine width visually matches expected thickness for 40 vs 80 page books
- All captions sit within their correct zones
- SVG overlays appear above photos
- No console errors on cover render
- Changing `bgColor` in the config bar updates the cover live
