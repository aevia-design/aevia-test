# Plan 10-02 — Caption Text Editor Toolbar

## Objective
Add a floating toolbar that appears above any focused caption `contenteditable` in the template engine. Controls: font family, font size stepper, text alignment, line spacing stepper. Per-caption styling only — no per-selection formatting. Direct inline style manipulation; no `execCommand`.

## Context
- @pages/template-engine.html — captions are `contenteditable` divs; styles currently set at render time from template-data defaults
- @assets/Template_Scribble/template-data.js — caption definitions include `fontFamily`, `sizePt`, `align`, `lineHeight` per slot
- @assets/fonts/ — lists available font files; toolbar font picker must only offer fonts present here

## Architecture decisions (follow exactly)
- Toolbar is a single shared DOM element (`id="caption-toolbar"`), absolutely positioned
- On `focus` of any `.caption-edit`: position toolbar above that element, populate controls from its current inline styles
- On `blur` of caption: hide toolbar (with 150ms delay to allow toolbar button clicks to register)
- Toolbar does NOT re-render the spread — it writes styles directly to the focused element
- Save: on every toolbar control change, update `window.bookCaptions[spreadIndex][side][slotKey].style` object (add a `style` property alongside existing `text`)
- On spread re-render, apply saved `.style` overrides on top of template defaults

## Tasks

### Task 1 — Toolbar HTML + CSS
Add one toolbar `<div id="caption-toolbar">` to the page body (outside spread rows). Hidden by default (`display:none`).

Controls in order:
1. **Font family** — `<select id="tb-font">`. Options: read from `window.AVAILABLE_FONTS` (define this as a const listing fonts in `assets/fonts/` — e.g. `['Lato', 'Playfair Display', 'FirstTimeWriting!']`). Display name = filename without extension.
2. **Font size** — `<button>−</button> <span id="tb-size">12</span>pt <button>+</button>`. Step 1pt. Range 6–72pt.
3. **Alignment** — three buttons: `⬛` left / `⬛` center / `⬛` right. Active state highlights current alignment.
4. **Line spacing** — `<button>−</button> <span id="tb-lh">1.1</span> <button>+</button>`. Step 0.05. Range 0.8–2.5.

Toolbar style: white background, 1px border (`var(--border)`), border-radius 6px, shadow `0 2px 8px rgba(0,0,0,0.12)`, padding 6px 10px, gap 12px between control groups. Font size 13px. Position: `position:fixed`, `z-index:1000`.

### Task 2 — Focus/blur wiring
On every `.caption-edit` `focus` event:
- Read current inline styles from element (font-family, font-size, text-align, line-height)
- Populate toolbar controls to match
- Store reference to focused element in `window._activeCaptionEl`
- Position toolbar: `getBoundingClientRect()` of focused element → place toolbar 8px above it, left-aligned, clamped to viewport edges
- Show toolbar

On `blur` (with 150ms delay):
- If `document.activeElement` is inside `#caption-toolbar`, cancel hide
- Otherwise hide toolbar and clear `window._activeCaptionEl`

### Task 3 — Control → style application
Each control change immediately:
1. Writes the style to `window._activeCaptionEl` as inline style
2. Updates the saved `style` object in `window.bookCaptions`

Mapping:
- Font select → `element.style.fontFamily = value`
- Size ± → parse current `font-size` from element, ±1pt, clamp 6-72, apply as `element.style.fontSize = N + 'pt'`
- Align buttons → `element.style.textAlign = 'left'|'center'|'right'`; update active button highlight
- Line spacing ± → parse current `line-height`, ±0.05, clamp 0.8-2.5, apply as `element.style.lineHeight = N`

### Task 4 — Persist through re-render
In `renderSpread()` (wherever caption elements are created), after setting default styles from template-data, check if `window.bookCaptions[spreadIndex][side][slotKey].style` exists. If yes, apply each property as an inline style override.

This ensures toolbar changes survive spread re-renders (e.g. after drag-drop or spread type swap).

## Verify
- Click a caption: toolbar appears above it, controls reflect current styles
- Change font: caption text changes font immediately
- Size +/− buttons change font size in 1pt steps, clamped correctly
- Alignment buttons toggle and highlight active state
- Line spacing steps work in 0.05 increments
- Toolbar stays visible while clicking its controls (blur delay working)
- After dragging a photo (re-render): toolbar-set styles are preserved on that caption
- FP text panels (Birthday wishes, Funny words) also respond to toolbar — they are contenteditable too
- No console errors
