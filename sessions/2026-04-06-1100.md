## Key decisions made
- Dashboard uses Firebase Web SDK v10 via CDN `<script type="module">` (not npm) — matches plain HTML/no-build-tool stack
- Dashboard password stored as `const PASS` in dashboard.html — not a secret per se, just keeps casual visitors out. Needs to be changed from default `aevia2026` before going live
- Firebase `apiKey` in frontend code is not a security risk — real security comes from Firestore rules
- Firestore order document ID = order number (e.g. `AEV-001`) for easy lookup
- Security hook uses Node.js (not Python — not available on this machine)
